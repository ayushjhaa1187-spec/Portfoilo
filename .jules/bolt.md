## 2024-10-24 - API Response Contract Changes
**Learning:** Altering API payloads (e.g., using `.select('-content')` or `.select('-fullDescription')`) without explicit frontend verification of field usage is considered a breaking change. While `.select()` reduces payload and memory overhead effectively, removing fields from existing REST API endpoints modifies the API contract and carries significant risk of breaking the application, regardless of current mock data setups.
**Action:** When optimizing read-only endpoints, prioritize database-level optimizations like adding indexes (e.g., `index: true` for frequently filtered fields like `category`) which speed up lookups without changing the response structure. If modifying API responses, only do so for newly built endpoints or ensure 100% frontend safety before proceeding.
