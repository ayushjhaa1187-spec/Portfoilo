## 2026-03-16 - Prevent Timing Attacks in Admin Auth
**Vulnerability:** Admin endpoint `POST /api/projects` was unauthenticated and vulnerable to unauthorized access. Implementing simple string comparison for `x-api-key` validation makes the endpoint susceptible to timing attacks, allowing attackers to guess the correct key character by character based on response time differences.
**Learning:** Comparing sensitive strings (like API keys, tokens, or passwords) character by character using simple `===` exposes execution time variations because the comparison stops at the first mismatched character. Normalizing input sizes via cryptographic hashes ensures the inputs to `timingSafeEqual` are always the same length, preventing the comparison function from failing prematurely on length discrepancies.
**Prevention:** Use `crypto.timingSafeEqual` along with `crypto.createHash('sha256')` to normalize lengths before performing the comparison, ensuring constant-time execution regardless of input validity.
