## 2024-05-18 - Mass Assignment and Internal Data Leakage in Contact API
**Vulnerability:** The POST `/api/contact` route directly passed `req.body` to the Mongoose constructor (`new Contact(req.body)`) and blindly echoed the saved document back to the client (`res.json({ msg: '...', contact })`).
**Learning:** This exposes a mass assignment vulnerability, as an attacker could inject unexpected fields (e.g., `status: 'read'`) into `req.body` that would be saved to the database. Additionally, returning the complete Mongoose document leaked internal database structures like `__v` and `_id`, which shouldn't be exposed on this route.
**Prevention:** Always explicitly destructure expected fields from user input before passing them to database creation methods (e.g., `const { name, email, subject, message } = req.body; new Contact({ name, email, subject, message });`). Do not return whole unsanitized database documents in responses; explicitly define success response payloads.
