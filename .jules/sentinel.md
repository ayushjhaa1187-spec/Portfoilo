## 2024-05-24 - Mass Assignment & Data Leakage in API Routes
**Vulnerability:** The `POST /api/contact` endpoint directly passed `req.body` to the `Contact` Mongoose model constructor (`new Contact(req.body)`). This allowed for mass assignment vulnerabilities where attackers could pass fields like `status` or `createdAt` to overwrite server-side values. Additionally, the endpoint echoed back the entire saved Mongoose document in the response (`{ msg: 'Message sent successfully', contact }`), potentially leaking internal fields or object structures to unauthorized clients.
**Learning:** Mongoose models do not automatically filter fields during initialization unless explicitly defined, and saving an object echoes back all properties, including internal database IDs or metadata.
**Prevention:** Always explicitly destructure incoming data from `req.body` to only allow expected fields (e.g., `const { name, email, subject, message } = req.body`) and avoid echoing raw database documents back in API responses unless specifically needed and cleansed.
