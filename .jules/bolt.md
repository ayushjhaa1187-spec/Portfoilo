## 2024-05-24 - Duplicate Layout Components in Next.js App Router
**Learning:** Next.js App Router automatically wraps page components (`page.tsx`) with the nearest layout component (`layout.tsx`). If layout components like `Navbar` and `Footer` are included in both `layout.tsx` and a specific `page.tsx` (like the homepage), they will be rendered twice. This creates unnecessary DOM nodes, significantly increases React hydration time, and harms overall frontend performance.
**Action:** When working with the Next.js App Router, always ensure that global layout components (like navigation bars and footers) are only rendered in `layout.tsx` files, and not duplicated inside individual `page.tsx` files unless explicitly required for a completely distinct layout (and in that case, route groups should be used).
