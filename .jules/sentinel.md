## 2024-03-12 - Secure API Key Comparison
**Vulnerability:** Missing authentication on the `POST /api/projects` endpoint allowed anyone to create projects. Additionally, when comparing API keys, a simple `===` or `!==` comparison can leak the length or matching prefix of the secret via timing attacks.
**Learning:** `crypto.timingSafeEqual` prevents timing attacks by taking a constant amount of time regardless of how many characters match. However, it requires both buffers to be the exact same length. If lengths differ, it throws an error. Hashing both the provided and expected secrets first (e.g., using `crypto.createHash('sha256')`) normalizes their lengths securely before the comparison.
**Prevention:** Always use `crypto.timingSafeEqual` for comparing secrets like API keys or signatures. Hash the strings first to safely normalize their lengths and avoid algorithmic DoS from slow key derivation functions like `scryptSync` when simple symmetric comparison is needed. Add authentication middleware (`auth.js`) to protect sensitive administrative endpoints.
