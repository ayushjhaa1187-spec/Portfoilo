## 2024-03-15 - [Mass Assignment and Data Leak in Mongoose Routes]
**Vulnerability:** The POST `/api/contact` endpoint directly instantiated a Mongoose model using `req.body` (`new Contact(req.body)`) and then returned the raw Mongoose document in the response (`res.json({ msg: '...', contact })`). This created a mass assignment vulnerability (allowing attackers to set internal fields like `status` or `createdAt`) and an information disclosure vulnerability (leaking database IDs, `__v`, and potentially other internal properties).
**Learning:** This is a recurring pattern in the codebase. Mongoose models are directly hydrated from user input and returned verbatim. It bypasses any schema-level protections unless explicitly validated or stripped.
**Prevention:** Always explicitly destructure and whitelist allowed fields from `req.body` before passing them to a model constructor or update method. Never echo raw Mongoose documents back to the client; always return explicit success messages or DTOs (Data Transfer Objects) that contain only safe data.
