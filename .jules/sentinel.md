## 2024-05-18 - Safe String Comparison for Auth Tokens
**Vulnerability:** Comparing API keys securely without introducing algorithmic DoS vectors. `crypto.timingSafeEqual` throws an error on length mismatch (a potential timing leak), and `crypto.scryptSync` used for hashing is slow and synchronous, making the server vulnerable to algorithmic DoS attacks if hammered.
**Learning:** Comparing unknown length secrets securely is tricky. Standard secure string comparison techniques (like hashing before comparison to equalize lengths) can be problematic in Node.js if the hashing algorithm is synchronously expensive (like `scryptSync` or `pbkdf2Sync`).
**Prevention:** Use a fast, non-blocking hash like `crypto.createHash('sha256')` to normalize the lengths of both the provided token and expected secret into `Buffer`s before comparing them with `crypto.timingSafeEqual`. This avoids timing leaks during length comparison and is not computationally expensive enough to be used as a DoS vector.
