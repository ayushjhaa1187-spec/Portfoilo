## 2024-03-18 - Prevent Algorithm DoS and Timing Attacks
**Vulnerability:** Synchronous algorithms like `crypto.scryptSync` can be used to cause an algorithmic denial of service when comparing strings or processing large request payloads due to high CPU usage. Typical direct string comparisons for keys might lead to timing attacks. Using `crypto.timingSafeEqual` prevents timing attacks, but it requires both buffers to have the exact same length.
**Learning:** Hashing variables using SHA256 before applying `crypto.timingSafeEqual` is a secure and efficient way to standardize buffer lengths for comparison, avoiding the overhead of heavy key-stretching algorithms that aren't necessary for high-entropy API key verification, while still protecting against timing attacks.
**Prevention:** When validating high-entropy secrets like API keys where dictionary attacks are less relevant, use `crypto.createHash('sha256').update(string).digest()` to normalize buffer lengths before feeding them into `crypto.timingSafeEqual`.
