## 2024-05-20 - [Timing-Safe API Key Verification]
**Vulnerability:** The `POST /api/projects` endpoint was missing authentication, and when implementing custom API key middleware, standard string comparison (`===`) is vulnerable to timing attacks (where an attacker can infer the key character-by-character based on response time differences).
**Learning:** Node.js provides `crypto.timingSafeEqual` for constant-time comparisons, but it requires both inputs to be `Buffer` or `TypedArray` objects of the exact same length. If the lengths differ, it throws an error instead of returning false, which can lead to a Denial of Service.
**Prevention:** Always normalize the lengths of secrets before using `crypto.timingSafeEqual` by hashing both the expected secret and the provided secret using a secure algorithm like SHA-256 (`crypto.createHash('sha256').update(value).digest()`). This ensures both buffers are identical in length (32 bytes) regardless of input size, preventing length-mismatch errors and ensuring a constant-time comparison.
